## 4. ARCHITECTURE — HOW THE BUILD WORKS

```
Step 1: build_manbearpig_v7.py connects to litigation_context.db
           |
Step 2: 12-phase extraction queries 10 major tables
           |
Step 3: gen_nodes() creates ~2,500 node objects from extracted data
           |
Step 4: gen_links() creates ~2,000 link objects connecting nodes
           |
Step 5: KRAKEN overlay merged (adds ~28 nodes, ~38 links)
           |
Step 6: Nodes/links JSON-serialized and EMBEDDED into HTML template
         (string replacement: __NODES__, __LINKS__, __LAYERS__, __STATS__)
           |
Step 7: HTML written to 12_WORKSPACE/THEMANBEARPIG_v7/THEMANBEARPIG_v7.html
           |
Step 8: graph_data_v7.json written (metadata only — node/link counts, stats)
           |
Step 9: PyInstaller bundles adversary_blueprint.py + HTML + JSON into .exe
```

### KEY INSIGHT: Data is IN the HTML, not in JSON
The builder embeds the full node/link arrays directly into the HTML via:
```python
html = HTML  # 800+ line HTML template string in the builder
html = html.replace('__NODES__', json.dumps(nodes, default=str, ensure_ascii=False))
html = html.replace('__LINKS__', json.dumps(links, default=str, ensure_ascii=False))
html = html.replace('__LAYERS__', json.dumps(LAYER_META))
html = html.replace('__STATS__', json.dumps(D['stats']))
html = html.replace('__SEP_DAYS__', str(SEP_DAYS))
```
The `graph_data_v7.json` file only stores metadata for reference.

## 9. KNOWN ISSUES & FIXES

### Issue 1: cp1252 Encoding Crash (HIGH RISK)
The builder uses emoji characters in `LAYER_META` (lines 25-44). If the Windows
console is cp1252, `print()` calls with emoji will crash with:
```
UnicodeEncodeError: 'charmap' codec can't encode characters
```

**Fix:** Either:
- Set `$env:PYTHONUTF8 = "1"` before running (preferred)
- Or add a `_safe()` helper at the top of the builder:
```python
def _safe(text):
    if isinstance(text, str):
        return text.encode('ascii', 'replace').decode('ascii')
    return str(text)
```
And wrap all print statements that output emoji/unicode. The print statements
at lines 76, 82, 84, 91, 93, 100, etc. may need wrapping.

### Issue 2: Hardcoded DB Path
Line 17: `DB = r"C:\Users\andre\LitigationOS\litigation_context.db"`
This is correct for Andrew's machine. If porting, change this line.

### Issue 3: THEMANBEARPIG.spec Missing Icon
The current spec (line 19-38) has NO `icon=` parameter. To add a custom icon:
```python
exe = EXE(
    ...
    icon='blueprint.ico',  # Add this line
    ...
)
```
The `blueprint.ico` file exists in the blueprint_build directory.

### Issue 4: graph_data_v7.json Is Metadata Only
The JSON file (lines 1341-1345) only saves:
```python
{"version": VERSION, "nodes": len(nodes), "links": len(links),
 "stats": D['stats'], "layers": [l['id'] for l in LAYER_META],
 "renderer": "PixiJS 7.3.3 WebGL2"}
```
This is BY DESIGN. The actual graph data lives in the HTML. Don't try to make
the exe load from JSON — the HTML already has everything embedded.

### Issue 5: PyInstaller spec bundles BOTH files
Line 8 of the spec:
```python
datas=[('...THEMANBEARPIG_v7.html', '.'), ('...graph_data_v7.json', '.')]
```
The JSON is bundled for reference but the launcher only loads the HTML file.
