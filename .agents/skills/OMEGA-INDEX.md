## Routing Decision Tree

```
REQUEST RECEIVED
│
├─ Is it litigation / legal / evidence / filing / court?
│   └─ YES → OMEGA-LITIGATION-SUPREME (ALWAYS — the unified system)
│       │
│       ├─ Need evidence processing specifically?
│       │   └─ SUPREME handles it (M1), but OMEGA-EVIDENCE has deeper module detail
│       │
│       ├─ Need legal research specifically?
│       │   └─ SUPREME handles it (M3), but OMEGA-RESEARCH has deeper module detail
│       │
│       └─ Need filing / motion / brief?
│           └─ SUPREME handles it (M4 + M9) — the primary filing engine
│
├─ Is it system design / agent architecture / orchestration?
│   └─ YES → OMEGA-ARCHITECT (meta agent)
│       │
│       ├─ Agent design → OMEGA-AGENT-ARCHITECT
│       ├─ Workflow / dispatch → OMEGA-ORCHESTRATOR
│       └─ MCP servers / tools → OMEGA-MCP
│
├─ Is it coding / data / security?
│   └─ YES → OMEGA-ENGINEER (meta agent)
│       │
│       ├─ Python / TypeScript code → OMEGA-CODE
│       ├─ SQL / database / schema → OMEGA-DATA
│       └─ Security audit / PII → OMEGA-SECURITY
│
├─ Is it maintenance / health / docs?
│   └─ YES → OMEGA-SENTINEL (meta agent)
│       │
│       ├─ Memory / recall → OMEGA-MEMORY
│       ├─ Git / CI / deploy / backup → OMEGA-DEVOPS
│       └─ Documentation / reports → OMEGA-WRITING
│
├─ Is it specialized (transcript analysis, form finding, redaction)?
│   └─ YES → Legacy agent (see agent-activation.instructions.md)
│       │
│       ├─ Hearing transcripts → transcript-analyzer
│       ├─ Michigan court forms → court-form-finder
│       ├─ PII redaction → redaction-agent
│       ├─ Filing QA → pre-filing-qa
│       └─ Complex legal phases → legal-phase-indexer
│
└─ None of the above?
    └─ Start with OMEGA-LITIGATION-SUPREME if legal-adjacent
    └─ Start with OMEGA-ENGINEER if technical
    └─ Ask for clarification if truly ambiguous
```
