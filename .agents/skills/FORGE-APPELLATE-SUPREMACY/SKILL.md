```python
from dataclasses import dataclass, field
from typing import List

@dataclass
class BriefSection:
    name: str
    required: bool = True
    page_budget: int = 0
    content: str = ""

@dataclass
class COABrief:
    case_no: str
    lower_court_no: str
    appellant: str
    appellee: str
    sections: List[BriefSection] = field(default_factory=list)

    def total_budget(self) -> int:
        return sum(section.page_budget for section in self.sections)

brief = COABrief(
    case_no="366810",
    lower_court_no="2024-001507-DC",
    appellant="Andrew James Pigors (pro se)",
    appellee="Emily A. Watson",
    sections=[
        BriefSection("Table of Contents", page_budget=2),
        BriefSection("Index of Authorities", page_budget=3),
        BriefSection("Jurisdictional Statement", page_budget=2),
        BriefSection("Questions Presented", page_budget=2),
        BriefSection("Statement of Facts", page_budget=10),
        BriefSection("Argument I", page_budget=10),
        BriefSection("Argument II", page_budget=10),
        BriefSection("Argument III", page_budget=8),
        BriefSection("Relief Requested", page_budget=1),
        BriefSection("Signature and Service", page_budget=1),
    ],
)

assert brief.total_budget() <= 50, "MCR 7.212 page budget exceeded"
for section in brief.sections:
    print(f"{section.name}: {section.page_budget} pages")
```

## AS2: Standard of Review Analyzer

### Purpose

### Design Pattern
Issue-classification engine with precedent-backed review labels and burden-aware issue decomposition.

### Detailed Description
- AS2 stops weak briefs from dying at the opening sentence of the argument section. In Michigan custody appeals, the standard of review is not ornamental; it defines what the Court of Appeals is allowed to do with the record.
- This module decomposes each question presented into factual findings, legal conclusions, discretionary rulings, preservation posture, and requested remedy, then assigns the correct review label to each layer.
- For COA Case No. 366810, AS2 expects recurring custody standards: great-weight review for findings, abuse-of-discretion review for dispositional choices, and clear legal error or de novo review for legal interpretation depending on issue framing.
- It uses Fletcher v Fletcher, Pierron v Pierron, and Vodvarka v Grasmeyer as recurring anchors while still allowing issue-specific authority updates from AS3.
- The result is a standards matrix that flows directly into brief headings, motion introductions, and oral-argument prep.

### Key Operations
1. Ingest the issue list from AS1 or AS7.
2. Tokenize each issue into factual, legal, discretionary, preservation, and remedy components.
3. Search authority chains for controlling Michigan standards-of-review cases.
4. Map factual best-interest findings to great-weight or clear-error style review depending on the cited authority and issue context.
5. Map custody disposition choices to abuse-of-discretion review anchored by Fletcher.
6. Map statutory interpretation, jurisdiction, and rule-construction issues to de novo review.
7. Surface mixed questions that require nested standards within one argument section.
8. Attach the selected review label, lead authority, and required proof language to each issue.

### Brief-to-Filing Pipeline
1. **AS7** locks the forum, vehicle, deadline, and TrueFiling route for the issue set.
