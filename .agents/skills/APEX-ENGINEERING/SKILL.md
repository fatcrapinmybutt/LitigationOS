## AE1: Full-Stack Application Genesis

### Purpose
Generate an entire production application from a specification, not by stapling
frontend code to backend code after the fact, but by deriving user flows,
contracts, persistence, tests, and deployment as one coordinated birth event.

### Design Pattern
**Spec Graph → Contract Spine → Runtime Mesh → Verification Envelope**

### APEX Design Axioms
- Start from domain verbs and invariants, never from framework folders.
- Generate shared contracts before components or endpoints to prevent drift.
- Model failure states during system birth, not as post-launch repairs.
- Provision observability and health endpoints in the first generation pass.
- Treat migrations and seeding as part of creation, not postscript chores.
- Ship deployable slices whose tests already reflect real production topology.

### Emergent Behaviors Unique to APEX
- Creates UI forms, backend handlers, schemas, and tests from the same invariant source, which a frontend-only or backend-only forge cannot do alone.
- Optimizes the initial application structure based on future migration cost, not just present coding convenience.
- Generates contract-safe client and server code simultaneously, preserving exact validation semantics across Python and TypeScript.
- Builds deployment-aware application boundaries so local module decisions already respect container and health-check constraints.
- Chooses state ownership boundaries using both database write patterns and frontend reactivity needs.
- Synthesizes a runnable baseline with meaningful observability, not just scaffolding files.

### Operating Protocol
1. Interpret the product request as a graph of actors, workflows, invariants, and operational risks.
2. Derive bounded contexts and define cross-context contract edges before writing framework code.
3. Generate persistence strategy and cache boundaries from workflow latency and consistency needs.
4. Emit backend endpoints and frontend mutations from the same contract definitions.
5. Bind health, telemetry, and feature flags into the runtime shell at generation time.
6. Produce the primary verification envelope: unit, integration, contract, and smoke tests.
7. Assemble deployment metadata so the app can be run, tested, and shipped immediately.

### Python Example
```python
from __future__ import annotations

from collections.abc import AsyncIterator, Callable
from contextlib import asynccontextmanager
from dataclasses import dataclass, field
from typing import Any
from uuid import UUID, uuid4

from fastapi import Depends, FastAPI, HTTPException
from pydantic import BaseModel, ConfigDict, Field


@dataclass(slots=True)
class EntitySpec:
    name: str
    storage: str
    searchable_fields: tuple[str, ...]
    unique_fields: tuple[str, ...]


@dataclass(slots=True)
class ApplicationSpec:
    service_name: str
    entities: list[EntitySpec]
    health_checks: tuple[str, ...] = ("db", "redis", "queue")
    feature_flags: dict[str, bool] = field(default_factory=dict)


class ProjectCreate(BaseModel):
    model_config = ConfigDict(extra="forbid", str_strip_whitespace=True)
    name: str = Field(min_length=3, max_length=120)
    slug: str = Field(pattern=r"^[a-z0-9-]+$")
    owner_email: str


class ProjectRead(ProjectCreate):
    id: UUID


class InMemoryProjectRepository:
    def __init__(self) -> None:
        self._rows: dict[str, ProjectRead] = {}

    async def create(self, payload: ProjectCreate) -> ProjectRead:
        if payload.slug in self._rows:
            raise ValueError("slug already exists")
        row = ProjectRead(id=uuid4(), **payload.model_dump())
        self._rows[payload.slug] = row
        return row

    async def get(self, slug: str) -> ProjectRead | None:
        return self._rows.get(slug)


class UnitOfWork:
    def __init__(self, projects: InMemoryProjectRepository) -> None:
        self.projects = projects

    async def __aenter__(self) -> "UnitOfWork":
        return self

    async def __aexit__(self, exc_type: Any, exc: Any, tb: Any) -> None:
        if exc:
            return

    async def commit(self) -> None:
        return


@asynccontextmanager
async def build_runtime(spec: ApplicationSpec) -> AsyncIterator[dict[str, Any]]:
    state = {
        "spec": spec,
        "projects": InMemoryProjectRepository(),
        "metrics": {"projects_created": 0},
    }
    yield state


def create_app(spec: ApplicationSpec) -> FastAPI:
    app = FastAPI(title=spec.service_name, version="1.0.0")

    @asynccontextmanager
    async def lifespan(_: FastAPI) -> AsyncIterator[None]:
        async with build_runtime(spec) as state:
            app.state.runtime = state
            yield

    app.router.lifespan_context = lifespan

    def get_uow() -> UnitOfWork:
        return UnitOfWork(app.state.runtime["projects"])

    @app.get("/health")
    async def health() -> dict[str, Any]:
        return {
            "service": spec.service_name,
            "checks": {name: "ok" for name in spec.health_checks},
            "feature_flags": spec.feature_flags,
        }

    @app.post("/projects", response_model=ProjectRead, status_code=201)
    async def create_project(
        payload: ProjectCreate,
        uow: UnitOfWork = Depends(get_uow),
    ) -> ProjectRead:
        async with uow:
            try:
                project = await uow.projects.create(payload)
            except ValueError as exc:
                raise HTTPException(status_code=409, detail=str(exc)) from exc
            await uow.commit()
            app.state.runtime["metrics"]["projects_created"] += 1
            return project

    @app.get("/projects/{slug}", response_model=ProjectRead)
    async def read_project(
        slug: str,
        uow: UnitOfWork = Depends(get_uow),
    ) -> ProjectRead:
        async with uow:
            project = await uow.projects.get(slug)
            if project is None:
                raise HTTPException(status_code=404, detail="project not found")
            return project

    return app


spec = ApplicationSpec(
    service_name="atlas-control-plane",
    entities=[EntitySpec("project", "postgresql", ("name", "slug"), ("slug",))],
    feature_flags={"billing": True, "audit_log": True},
)

app = create_app(spec)
```

## AE2: Polyglot Code Transmutation

### Purpose
Translate systems between Python, TypeScript, Go, Rust, and other targets by first
preserving semantics in a neutral intermediate representation, then emitting
idiomatic code with compatibility adapters and contract continuity.

### Design Pattern
**Semantic IR → Idiom Mapper → Compatibility Adapter → Verification Mirror**

### APEX Design Axioms
- Never translate syntax directly when semantics can be preserved in a neutral IR.
- Model side effects explicitly so translated code can keep transactional intent.
- Retain boundary contracts first; implementation details may vary by language idiom.
- Generate migration adapters to allow partial rollout across polyglot estates.
- Pair every transmutation with test mirror generation to prove semantic parity.
- Emit code that feels native in the target language, not mechanically ported.

### Emergent Behaviors Unique to APEX
- Allows a Python service to become a TypeScript edge worker or Rust service without losing invariants, because the APEX understands runtime, type, database, and test consequences together.
- Generates both target implementation and bridge adapters, enabling phased migrations rather than risky big-bang rewrites.
- Preserves test surfaces across languages, which language-specific forges cannot guarantee on their own.
- Translates data semantics while re-optimizing for the target runtime's concurrency and serialization model.
- Uses database and performance context from other modules to avoid 'correct but slow' ports.
- Produces forward-compatible interfaces that minimize disruption to clients during transmutation.

### Operating Protocol
1. Parse source modules into a semantic IR that captures models, effects, contracts, and ownership.
2. Normalize domain primitives so the same business meaning can survive target-language differences.
3. Map semantics to target idioms: async primitives, type systems, memory ownership, or module boundaries.
4. Generate compatibility adapters so old callers and new implementations can coexist safely.
5. Emit mirrored test scaffolds that prove behavioral equivalence after translation.
6. Re-run performance and persistence heuristics so the translated code fits its new runtime reality.
7. Package the transmutation as a migration wave, not an isolated code dump.

### Python Example
```python
from __future__ import annotations

from dataclasses import dataclass, field
from typing import Literal


Primitive = Literal["string", "integer", "float", "boolean", "datetime", "uuid"]


@dataclass(slots=True)
class FieldIR:
    name: str
    primitive: Primitive
    required: bool = True
    repeated: bool = False


@dataclass(slots=True)
class FunctionIR:
    name: str
    parameters: list[FieldIR]
    returns: FieldIR
    effects: tuple[str, ...] = ()


@dataclass(slots=True)
class ModuleIR:
    language: str
    models: dict[str, list[FieldIR]] = field(default_factory=dict)
    functions: list[FunctionIR] = field(default_factory=list)


def emit_python_model(name: str, fields: list[FieldIR]) -> str:
    body = ["class %s(BaseModel):" % name]
    for field in fields:
        optional = " | None" if not field.required else ""
        container = f"list[{field.primitive}{optional}]" if field.repeated else f"{field.primitive}{optional}"
        body.append(f"    {field.name}: {container}")
    return "\n".join(body)


def emit_typescript_model(name: str, fields: list[FieldIR]) -> str:
    rows = [f"export interface {name} {{"]
    for field in fields:
        optional = "?" if not field.required else ""
        scalar = {
            "string": "string",
            "integer": "number",
            "float": "number",
            "boolean": "boolean",
            "datetime": "string",
            "uuid": "string",
        }[field.primitive]
        target = f"{scalar}[]" if field.repeated else scalar
        rows.append(f"  {field.name}{optional}: {target};")
    rows.append("}")
    return "\n".join(rows)


def emit_go_handler(fn: FunctionIR) -> str:
    args = ", ".join(f"{field.name} string" for field in fn.parameters)
    return "\n".join(
        [
            f"func {fn.name.title()}({args}) string {{",
            '    // generated from IR: side effects remain explicit',
            f'    return "{fn.returns.name}"',
            "}",
        ]
    )


def transmute(ir: ModuleIR, target: str) -> str:
    if target == "python":
        model_blocks = [emit_python_model(name, fields) for name, fields in ir.models.items()]
        return "\n\n".join(model_blocks)

    if target == "typescript":
        model_blocks = [emit_typescript_model(name, fields) for name, fields in ir.models.items()]
        return "\n\n".join(model_blocks)

    if target == "go":
        return "\n\n".join(emit_go_handler(fn) for fn in ir.functions)

    raise ValueError(f"unsupported target: {target}")


billing_ir = ModuleIR(
    language="neutral",
    models={
        "Invoice": [
            FieldIR("id", "uuid"),
            FieldIR("customer_id", "uuid"),
            FieldIR("amount_cents", "integer"),
            FieldIR("paid", "boolean"),
            FieldIR("issued_at", "datetime"),
        ]
    },
    functions=[
        FunctionIR(
            name="capture_payment",
            parameters=[FieldIR("invoice_id", "uuid"), FieldIR("actor_id", "uuid")],
            returns=FieldIR("receipt_id", "uuid"),
            effects=("audit_log", "outbox_event"),
        )
    ],
)

print(transmute(billing_ir, "python"))
print(transmute(billing_ir, "typescript"))
```

### TypeScript Example
```typescript
type Primitive = 'string' | 'integer' | 'float' | 'boolean' | 'datetime' | 'uuid';

interface FieldIR {
  name: string;
  primitive: Primitive;
  required: boolean;
  repeated?: boolean;
}

interface FunctionIR {
  name: string;
  parameters: FieldIR[];
  returns: FieldIR;
  effects: readonly string[];
}

interface ModuleIR {
  namespace: string;
  models: Record<string, FieldIR[]>;
  functions: FunctionIR[];
}

const scalarMap: Record<Primitive, string> = {
  string: 'string',
  integer: 'number',
  float: 'number',
  boolean: 'boolean',
  datetime: 'string',
  uuid: 'string',
};

function emitRustModel(name: string, fields: FieldIR[]): string {
  const rows = [`#[derive(Debug, Clone, serde::Serialize, serde::Deserialize)]`, `pub struct ${name} {`];
  for (const field of fields) {
    const base = scalarMap[field.primitive] === 'number' ? 'i64' : 'String';
    const ty = field.repeated ? `Vec<${base}>` : base;
    rows.push(`    pub ${field.name}: ${field.required ? ty : `Option<${ty}>`},`);
  }
  rows.push('}');
  return rows.join('\n');
}

function emitTypeScriptClient(fn: FunctionIR): string {
  const params = fn.parameters
    .map((field) => `${field.name}${field.required ? '' : '?'}: ${scalarMap[field.primitive]}`)
    .join(', ');

  return [
    `export async function ${fn.name}(${params}): Promise<${scalarMap[fn.returns.primitive]}> {`,
    `  const response = await fetch('/rpc/${fn.name}', {`,
    `    method: 'POST',`,
    `    headers: { 'content-type': 'application/json' },`,
    `    body: JSON.stringify({ ${fn.parameters.map((field) => field.name).join(', ')} }),`,
    `  });`,
    `  if (!response.ok) throw new Error('${fn.name} failed');`,
    `  const json = await response.json();`,
    `  return json.${fn.returns.name} as ${scalarMap[fn.returns.primitive]};`,
    `}`,
  ].join('\n');
}

function emitCompatibilityAdapter(source: ModuleIR, target: 'rust' | 'typescript') {
  if (target === 'rust') {
    return Object.entries(source.models)
      .map(([name, fields]) => emitRustModel(name, fields))
      .join('\n\n');
  }

  return source.functions.map(emitTypeScriptClient).join('\n\n');
}

const analyticsIR: ModuleIR = {
  namespace: 'analytics',
  models: {
    EventEnvelope: [
      { name: 'id', primitive: 'uuid', required: true },
      { name: 'name', primitive: 'string', required: true },
      { name: 'occurred_at', primitive: 'datetime', required: true },
      { name: 'tags', primitive: 'string', required: false, repeated: true },
    ],
  },
  functions: [
    {
      name: 'ingestEvent',
      parameters: [
        { name: 'eventId', primitive: 'uuid', required: true },
        { name: 'actorId', primitive: 'uuid', required: true },
      ],
      returns: { name: 'accepted', primitive: 'boolean', required: true },
      effects: ['queue', 'warehouse'],
    },
  ],
};

console.log(emitCompatibilityAdapter(analyticsIR, 'rust'));
console.log(emitCompatibilityAdapter(analyticsIR, 'typescript'));
```

```python
def select_engine(obj: DataObject) -> Engine:
    high_aggregation = any(pattern.read_shape == "aggregation" for pattern in obj.patterns)
    low_latency = any(pattern.latency_ms <= 5 for pattern in obj.patterns)
    flexible_shape = any(pattern.read_shape == "document" for pattern in obj.patterns)
    archival = all(pattern.retention_days > 365 for pattern in obj.patterns)

    if low_latency and obj.mutable:
        return "redis"
    if flexible_shape and obj.relationships <= 1:
        return "mongodb"
    if high_aggregation and obj.relationships >= 2:
        return "postgresql"
    if archival and obj.size_kb < 64:
        return "sqlite"
    return "mysql"


def placement_report(objects: list[DataObject]) -> dict[str, dict[str, object]]:
    report: dict[str, dict[str, object]] = {}
    for obj in objects:
        engine = select_engine(obj)
        report[obj.name] = {
            "engine": engine,
            "cache": engine not in {"redis", "sqlite"},
            "replication": engine in {"postgresql", "mysql", "mongodb"},
            "migration_mode": "expand-contract" if obj.mutable else "append-only",
        }
    return report
```

```typescript
interface PlacementDecision {
  engine: Engine;
  cache: boolean;
  replication: boolean;
  migrationMode: 'expand-contract' | 'append-only';
}

interface OrderSummary {
  orderId: string;
  customerId: string;
  totalCents: number;
  status: 'draft' | 'confirmed' | 'shipped';
  updatedAt: string;
}

interface Store<T> {
  get(key: string): Promise<T | null>;
  put(key: string, value: T): Promise<void>;
}

class CacheAsideStore<T> implements Store<T> {
  constructor(
    private readonly cache: Store<T>,
    private readonly source: Store<T>,
    private readonly ttlSeconds: number,
  ) {}

  async get(key: string): Promise<T | null> {
    const cached = await this.cache.get(key);
    if (cached) return cached;

    const loaded = await this.source.get(key);
    if (loaded) {
      await this.cache.put(key, loaded);
    }
    return loaded;
  }

  async put(key: string, value: T): Promise<void> {
    await this.source.put(key, value);
    await this.cache.put(key, value);
  }
}

function buildDecisionMatrix(entity: 'order' | 'session' | 'catalog'): PlacementDecision {
  switch (entity) {
    case 'session':
      return { engine: 'redis', cache: false, replication: false, migrationMode: 'expand-contract' };
    case 'catalog':
      return { engine: 'mongodb', cache: true, replication: true, migrationMode: 'append-only' };
    default:
      return { engine: 'postgresql', cache: true, replication: true, migrationMode: 'expand-contract' };
  }
}

export async function hydrateOrderSummary(
  orderStore: Store<OrderSummary>,
  cacheStore: Store<OrderSummary>,
  orderId: string,
): Promise<OrderSummary | null> {
  const store = new CacheAsideStore(cacheStore, orderStore, 30);
  const summary = await store.get(orderId);

  if (!summary) {
    return null;
  }

  const decision = buildDecisionMatrix('order');
  return {
    ...summary,
    status: decision.replication ? summary.status : 'draft',
  };
}
```

### Purpose

## AE5: Performance Singularity

### Purpose

### Operating Protocol
1. Define user-visible budgets and map them to component, API, and storage sub-budgets.
2. Instrument all relevant boundaries with a shared trace vocabulary.

```python
from __future__ import annotations

from contextlib import contextmanager
from dataclasses import dataclass, field
from statistics import mean
from time import perf_counter
from typing import Callable


@dataclass(slots=True)
class Sample:
    name: str
    duration_ms: float
    metadata: dict[str, object] = field(default_factory=dict)


@dataclass(slots=True)
class Budget:
    name: str
    threshold_ms: float


class BenchmarkSuite:
    def __init__(self) -> None:
        self.samples: list[Sample] = []

    @contextmanager
    def measure(self, name: str, **metadata: object):
        started = perf_counter()
        try:
            yield
        finally:
            elapsed = (perf_counter() - started) * 1000
            self.samples.append(Sample(name=name, duration_ms=elapsed, metadata=dict(metadata)))

    def report(self) -> dict[str, float]:
        grouped: dict[str, list[float]] = {}
        for sample in self.samples:
            grouped.setdefault(sample.name, []).append(sample.duration_ms)
        return {name: round(mean(values), 2) for name, values in grouped.items()}

    def assert_budgets(self, budgets: list[Budget]) -> None:
        report = self.report()
        violations = [
            f"{budget.name}: {report.get(budget.name, 0):.2f}ms > {budget.threshold_ms:.2f}ms"
            for budget in budgets
            if report.get(budget.name, 0) > budget.threshold_ms
        ]
        if violations:
            raise RuntimeError("budget violations: " + "; ".join(violations))


def benchmark_checkout(run_query: Callable[[], None], render_receipt: Callable[[], None]) -> dict[str, float]:
    suite = BenchmarkSuite()

    for _ in range(5):
        with suite.measure("db.checkout_query", layer="database"):
            run_query()
        with suite.measure("api.render_receipt", layer="backend"):
            render_receipt()

    suite.assert_budgets(
        [
            Budget("db.checkout_query", 25),
            Budget("api.render_receipt", 15),
        ]
    )
    return suite.report()


print(
    benchmark_checkout(
        run_query=lambda: sum(i for i in range(30_000)),
        render_receipt=lambda: "|".join(str(i) for i in range(2_000)),
    )
)
```

```typescript
  name: string;

type SpanRecord = {
  name: string;
  durationMs: number;
  layer: 'client' | 'edge' | 'api' | 'db';
};

class PerformanceLedger {
  private readonly rows: SpanRecord[] = [];

  async measure<T>(name: string, layer: SpanRecord['layer'], fn: () => Promise<T>): Promise<T> {
    const started = performance.now();
    try {
      return await fn();
    } finally {
      this.rows.push({ name, layer, durationMs: performance.now() - started });
    }
  }

  report() {
    return this.rows.reduce<Record<string, number>>((acc, row) => {
      const current = acc[row.name] ?? 0;
      acc[row.name] = Math.max(current, row.durationMs);
      return acc;
    }, {});
  }

  assertBudgets(budgets: Budget[]) {
    const report = this.report();
    const violations = budgets.filter((budget) => (report[budget.name] ?? 0) > budget.thresholdMs);
    if (violations.length > 0) {
      throw new Error(
        violations
          .map((budget) => `${budget.name}: ${(report[budget.name] ?? 0).toFixed(2)}ms`)
          .join(', '),
      );
    }
  }
}

  });

  ledger.assertBudgets([
    { name: 'client.initial_render', thresholdMs: 60 },
```

## AE6: Architecture Evolution Engine

### Purpose
Refactor, migrate, and modernize any codebase by turning legacy constraints into a
wave-based transformation program with compatibility adapters, rollback paths,
test gates, and performance-aware sequencing.

### Design Pattern
**Legacy Graph → Safe Waves → Adapter Mesh → Deletion Discipline**

### APEX Design Axioms
- Evolve in waves, not heroic rewrites.
- Every modernization step must have a rollback story before it has a merge story.
- Compatibility adapters buy time; deletion discipline prevents adapter bloat.
- Refactors that change contracts require mirrored tests and staged cutovers.
- Modernization is successful only when architecture becomes simpler to change afterward.
- Debt removal should reduce uncertainty, not merely rearrange files.

### Emergent Behaviors Unique to APEX
- Modernizes systems with awareness of types, runtime semantics, database migrations, and full-pyramid tests simultaneously.
- Creates phased plans that are safe for live systems because it knows where contracts, data, and user journeys are fragile.
- Decides when to rewrite, wrap, translate, or freeze a component based on evidence from all other modules.
- Pairs codemods with operational rollout logic, not just source transformations.
- Uses performance and persistence knowledge to sequence migrations in the least risky order.
- Turns legacy codebases into evolvable systems rather than cosmetically updated ones.

### Operating Protocol
1. Model the legacy system as a dependency graph of contracts, data stores, runtime seams, and risk zones.
2. Define safe waves: foundations, compatibility, cutover, cleanup.
3. Generate adapters and dual-write or dual-read strategies where boundaries must remain stable.
4. Bind every wave to explicit tests, metrics, and rollback criteria.
5. Execute cutovers only after parity, performance, and migration evidence are green.
6. Delete obsolete paths aggressively once confidence thresholds are met.
7. Re-run the generation and optimization models so the modernized system is easier to extend than the one it replaced.

```typescript
    id: 'cm-002',
    description: 'Move ad hoc DTO interfaces to shared contract package',
    wave: 'compatibility',
    matcher: /interface\s+\w+Dto\s+\{/g,
    replace: (source) => source.replace(/interface\s+(\w+)Dto\s+\{/g, 'type $1Dto = Contracts.$1; //'),
  },
];

export function applyCodemods(input: string) {
  return codemods.reduce(
    (current, task) => (task.matcher.test(current) ? task.replace(current) : current),
    input,
  );
}

export class LegacyBillingAdapter {
  async createInvoice(payload: { customerId: string; amountCents: number }) {
    return {
      invoiceId: crypto.randomUUID(),
      status: 'queued',
      payload,
    };
  }
}

export class ModernBillingPort {
  constructor(private readonly legacy: LegacyBillingAdapter) {}

  async issueInvoice(command: { customerId: string; amountCents: number }) {
    const result = await this.legacy.createInvoice(command);
    return {
      id: result.invoiceId,
      state: result.status === 'queued' ? 'pending' : 'issued',
      totalCents: command.amountCents,
    };
  }
}
```

1. **AE5** finds a cross-layer hotspot: slow query + oversized payload + expensive client render.
2. **AE3** redesigns the read model and cache policy.
3. **AE1** adjusts the API contract and component boundary to ship less data.
4. **AE4** adds regression and budget tests that lock in the win.
5. **AE6** packages the optimization into a safe migration wave with rollback.
6. The APEX turns a hotspot into a permanent architectural improvement, not a temporary patch.
