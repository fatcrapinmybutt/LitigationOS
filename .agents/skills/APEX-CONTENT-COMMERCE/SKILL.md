## AC1: Content Commerce Orchestrator

**Purpose:** Unify ideation, creation, API packaging, monetization, documentation, and distribution into one canonical operating flow.

**Design Pattern:** Signal Graph Orchestrator — one commerce brief becomes the shared state object for content, API, pricing, docs, and distribution decisions.

### Detailed Description

AC1 is the sovereign entry point for Content-Driven Commerce Intelligence. It does not treat editorial
planning, API product design, subscription packaging, and documentation as separate workstreams that happen
to share a launch date. Instead, it creates a canonical commerce brief that contains audience intent,
product promise, monetization targets, API capabilities, onboarding requirements, documentation obligations,
and channel narratives in one object graph. That brief is the thing every downstream module reads and
updates. The result is that content does not drift away from the product, pricing does not drift away from
usage reality, and documentation does not lag behind the API surface. AC1 is where the loop becomes closed
before the first asset is generated.

The APEX leap comes from how AC1 fuses the four FORGE systems. From FORGE-CONTENT-QUANTUM it inherits
ideation, audience segmentation, format selection, and optimization memory. From FORGE-SAAS-GENESIS it gains
plan logic, onboarding stages, retention checkpoints, and revenue intent. From FORGE-API-NEXUS it absorbs
contract modeling, usage envelopes, policy controls, and developer workflow framing. From FORGE-DOCS-APEX it
gains the discipline to make every feature, endpoint, and monetization rule explainable at the moment it is
conceived. None of those systems alone can preserve a single state model across the entire journey. AC1 can,
and that continuity is what makes the commerce loop autonomous rather than merely coordinated.

Operationally, AC1 decides sequence and dependency. A single topic cluster might justify a landing page, a
tutorial series, an interactive API starter, a changelog announcement, and a usage-based package with a free
quota. AC1 calculates that cascade upfront and attaches quality gates to each stage: relevance score before
drafting, readiness score before API exposure, compliance score before documentation publication, and launch
confidence before distribution. It also routes backward signals: poor trial-to-paid conversion can reopen
positioning, weak SDK adoption can reopen tutorial structure, and usage spikes can trigger pricing or rate-
limit redesign. This means every artifact is born knowing which downstream metric can force it to evolve.

The emergent behavior impossible without all four forges is holistic pipeline authorship. AC1 can say, with
one decision, that a new feature should ship as a metered API, launch with comparison content, explain
itself through interactive docs, and carry a specific growth hypothesis tied to activation and retention. A
content forge alone could publish. A SaaS forge alone could monetize. An API forge alone could expose
endpoints. A docs forge alone could explain them. AC1 is the only layer that can make those outputs mutually
reinforcing from the start, which is why the entire superskill inherits its defining capability from this
module.

### APEX-Only Behaviors

- Creates one canonical launch brief that every module can mutate without translation loss.
- Maps every content asset to a monetization hypothesis and every monetized feature to a documentation
  obligation.
- Uses downstream activation, retention, and usage signals as first-class rewrite triggers for
  upstream editorial decisions.
- Ensures API packaging, onboarding, and distribution sequencing are designed before production code
  and copy diverge.
- Turns roadmaps into executable cascades rather than disconnected backlogs.

### Key Operations

- Normalize product intent into a unified commerce brief with audience, feature, pricing, docs, and
  launch metadata.
- Compute stage dependencies across ideation, draft generation, API design, SDK production, pricing
  review, and multi-channel release.
- Emit quality gates for SEO readiness, API correctness, monetization safety, and documentation
  completeness.
- Route failure feedback from churn, low activation, weak conversion, or poor developer adoption back
  to the correct upstream stage.
- Version campaign, product, and documentation decisions as one portfolio of linked launch artifacts.
- Coordinate launch calendars so changelog, docs, social, blog, and email publish in controlled
  sequence.
- Surface revenue-at-risk when any content, API, or docs dependency slips.

### Code Example — Pipeline contract that preserves content, API, pricing, documentation, and distribution state in one graph

```python
from __future__ import annotations

from dataclasses import dataclass, field
from datetime import datetime, timezone
from enum import Enum
from typing import Iterable
import uuid


class Stage(str, Enum):
    IDEATION = "ideation"
    CREATION = "creation"
    API_PACKAGING = "api_packaging"
    MONETIZATION = "monetization"
    DOCUMENTATION = "documentation"
    DISTRIBUTION = "distribution"


@dataclass(slots=True)
class MetricTarget:
    name: str
    target: float
    owner: str


@dataclass(slots=True)
class CommerceBrief:
    brief_id: str
    audience: str
    promise: str
    api_capabilities: list[str]
    pricing_model: str
    content_formats: list[str]
    documentation_tracks: list[str]
    launch_channels: list[str]
    targets: list[MetricTarget] = field(default_factory=list)


@dataclass(slots=True)
class StageResult:
    stage: Stage
    status: str
    artifacts: dict[str, str]
    metrics: dict[str, float]
    timestamp: datetime


class ArtifactStore:
    def __init__(self) -> None:
        self._rows: dict[str, list[StageResult]] = {}

    def append(self, brief_id: str, result: StageResult) -> None:
        self._rows.setdefault(brief_id, []).append(result)

    def history(self, brief_id: str) -> list[StageResult]:
        return list(self._rows.get(brief_id, []))


class ContentCommerceOrchestrator:
    def __init__(self, store: ArtifactStore) -> None:
        self.store = store

    def _gate(self, stage: Stage, score: float) -> str:
        thresholds = {
            Stage.IDEATION: 0.70,
            Stage.CREATION: 0.75,
            Stage.API_PACKAGING: 0.80,
            Stage.MONETIZATION: 0.78,
            Stage.DOCUMENTATION: 0.82,
            Stage.DISTRIBUTION: 0.76,
        }
        return "ready" if score >= thresholds[stage] else "revise"

    def _emit_result(
        self,
        brief: CommerceBrief,
        stage: Stage,
        score: float,
        artifacts: dict[str, str],
    ) -> StageResult:
        result = StageResult(
            stage=stage,
            status=self._gate(stage, score),
            artifacts=artifacts,
            metrics={"quality_score": score},
            timestamp=datetime.now(timezone.utc),
        )
        self.store.append(brief.brief_id, result)
        return result

    def ideate(self, brief: CommerceBrief) -> StageResult:
        score = 0.84 if brief.api_capabilities and brief.content_formats else 0.63
        artifacts = {
            "content_brief": f"{brief.audience}:{brief.promise}",
            "launch_hypothesis": "api_usage_drives_subscription_upgrade",
        }
        return self._emit_result(brief, Stage.IDEATION, score, artifacts)

    def create(self, brief: CommerceBrief) -> StageResult:
        score = 0.81 if "tutorial" in brief.content_formats else 0.74
        artifacts = {
            "blog": "generated",
            "landing_page": "generated",
            "email_series": "generated",
        }
        return self._emit_result(brief, Stage.CREATION, score, artifacts)

    def package_api(self, brief: CommerceBrief) -> StageResult:
        score = 0.83 if len(brief.api_capabilities) >= 2 else 0.71
        artifacts = {
            "product_sku": "pro-api-metered",
            "rate_limit_policy": "starter-1000|growth-10000|scale-100000",
        }
        return self._emit_result(brief, Stage.API_PACKAGING, score, artifacts)

    def monetize(self, brief: CommerceBrief) -> StageResult:
        score = 0.80 if "usage-based" in brief.pricing_model else 0.77
        artifacts = {
            "billing_model": brief.pricing_model,
            "trial_offer": "14-day developer trial",
        }
        return self._emit_result(brief, Stage.MONETIZATION, score, artifacts)

    def document(self, brief: CommerceBrief) -> StageResult:
        score = 0.86 if "quickstart" in brief.documentation_tracks else 0.73
        artifacts = {
            "portal": "published",
            "sdk_samples": "generated",
            "changelog_entry": "drafted",
        }
        return self._emit_result(brief, Stage.DOCUMENTATION, score, artifacts)

    def distribute(self, brief: CommerceBrief) -> StageResult:
        score = 0.79 if len(brief.launch_channels) >= 3 else 0.68
        artifacts = {
            "channels": ",".join(brief.launch_channels),
            "campaign_id": uuid.uuid4().hex,
        }
        return self._emit_result(brief, Stage.DISTRIBUTION, score, artifacts)

    def run(self, brief: CommerceBrief) -> list[StageResult]:
        stages = (
            self.ideate,
            self.create,
            self.package_api,
            self.monetize,
            self.document,
            self.distribute,
        )
        return [stage(brief) for stage in stages]


brief = CommerceBrief(
    brief_id=uuid.uuid4().hex,
    audience="developer platform teams",
    promise="turn content performance into API revenue growth",
    api_capabilities=["analytics", "pricing_recommendation", "portal_generation"],
    pricing_model="hybrid usage-based subscription",
    content_formats=["blog", "landing_page", "tutorial", "social"],
    documentation_tracks=["quickstart", "api_reference", "migration_guide"],
    launch_channels=["docs", "blog", "email", "social"],
    targets=[
        MetricTarget("trial_to_paid", 0.18, "growth"),
        MetricTarget("activation_rate", 0.55, "product"),
        MetricTarget("content_to_signup", 0.07, "marketing"),
    ],
)

orchestrator = ContentCommerceOrchestrator(ArtifactStore())
history = orchestrator.run(brief)
for item in history:
    print(item.stage.value, item.status, item.metrics["quality_score"])
```

## AC2: API Product Designer

**Purpose:** Design API products as monetized developer experiences with contracts, tiers, rate limits, SDKs, samples, and interactive docs defined together.

**Design Pattern:** Contract-to-Offer Compiler — transform endpoint capabilities into priced, documented, and testable product packages.

### Detailed Description

AC2 exists because an API is not yet a product when it merely exposes endpoints. To become a product, the
surface must carry understandable value, clear constraints, reliable onboarding paths, predictable pricing,
usable samples, and a documentation experience that makes the first successful call feel inevitable. AC2
designs those dimensions simultaneously. It takes the technical graph of resources, methods, events, and
policies and converts it into a developer offer: named plans, usage envelopes, upgrade boundaries, SDK
targets, auth guidance, playground scenarios, and interactive reference narratives. The work is commercial
and technical at the same time.

This module only becomes possible at APEX because the four source forges contribute mutually dependent
logic. Content intelligence decides how the API should be framed and which use cases deserve landing pages,
comparison copy, and tutorials. SaaS intelligence decides how value should be captured through trials,
seats, overages, or metered packages. API intelligence decides how contracts, rate limits, versioning, and
gateways can support those offers without breaking trust. Documentation intelligence ensures that every tier
promise is accompanied by reference material, samples, migration notes, and supportable developer pathways.
AC2 composes those concerns into one product artifact rather than allowing monetization and usability to
fight each other.

AC2 also solves the classic distortion problem of API businesses: packaging decisions are often made far
away from developer experience. The pricing page promises capabilities that the docs do not clarify; the
docs demonstrate workflows that the lowest tier cannot actually complete; the SDKs hide advanced behavior
behind missing examples; and the rate-limit policy punishes evaluation traffic before trust is established.
AC2 treats those as design bugs, not copy mistakes. Its job is to ensure that the cheapest plan, the first
tutorial, the quickest sandbox experience, and the long-term scale path remain coherent across engineering,
marketing, and revenue functions.

The emergent behavior here is that AC2 can manufacture API businesses from capability graphs. It can propose
a starter tier because it knows which endpoints drive aha moments, recommend a pro tier because it sees
where monetizable value accumulates, and author the corresponding portal because it already understands the
teaching sequence. No single forge can hold that perspective. Only their fusion can produce an API offer
where surface design, business model, and documentation architecture converge into a single developer-facing
product.

### APEX-Only Behaviors

- Treats versioning, rate limits, and upgrade messaging as one pricing conversation rather than three
  separate governance tasks.
- Chooses SDK targets and sample languages based on acquisition channels, developer personas, and
  expected expansion revenue.
- Builds interactive documentation and pricing tables from the same product metadata, eliminating
  mismatch between docs and commerce.
- Transforms endpoint-level capability maps into named, teachable, marketable packages with clear
  onboarding paths.
- Uses content demand signals to decide which API workflows deserve premium features, tutorials, and
  landing pages first.

### Key Operations

- Cluster endpoints into jobs-to-be-done rather than exposing raw service boundaries as pricing plans.
- Design free, growth, and enterprise tiers with quota logic, SLA posture, support model, and
  expansion triggers.
- Attach auth models, example apps, and SDK generation targets to each plan.
- Generate interactive documentation requirements directly from productized use cases and contract
  groups.
- Select versioning and deprecation strategy aligned with revenue protection and migration simplicity.
- Recommend rate-limit envelopes that support evaluation without sacrificing unit economics or
  platform stability.
- Define upgrade narratives so documentation and pricing copy reinforce the same progression path.

### Code Example — API packaging engine that designs tiers, quotas, SDK strategy, and portal assets together

```python
from __future__ import annotations

from dataclasses import dataclass, field
from typing import Iterable


@dataclass(slots=True)
class Endpoint:
    name: str
    method: str
    path: str
    monetization_weight: int
    onboarding_value: int
    category: str


@dataclass(slots=True)
class RateLimitPolicy:
    requests_per_minute: int
    burst: int
    monthly_quota: int


@dataclass(slots=True)
class ProductTier:
    code: str
    monthly_price: int
    included_categories: list[str]
    policy: RateLimitPolicy
    support_model: str
    sdk_targets: list[str]


@dataclass(slots=True)
class PortalAsset:
    asset_type: str
    title: str
    audience: str
    related_tier: str


class ApiProductDesigner:
    def __init__(self, endpoints: Iterable[Endpoint]) -> None:
        self.endpoints = list(endpoints)

    def _categories_for(self, minimum_weight: int) -> list[str]:
        categories = {
            endpoint.category
            for endpoint in self.endpoints
            if endpoint.monetization_weight >= minimum_weight
        }
        return sorted(categories)

    def build_tiers(self) -> list[ProductTier]:
        starter = ProductTier(
            code="starter",
            monthly_price=0,
            included_categories=self._categories_for(1),
            policy=RateLimitPolicy(30, 10, 25_000),
            support_model="community",
            sdk_targets=["python", "typescript"],
        )
        growth = ProductTier(
            code="growth",
            monthly_price=199,
            included_categories=self._categories_for(2),
            policy=RateLimitPolicy(180, 60, 500_000),
            support_model="email",
            sdk_targets=["python", "typescript", "go"],
        )
        enterprise = ProductTier(
            code="enterprise",
            monthly_price=1499,
            included_categories=self._categories_for(3),
            policy=RateLimitPolicy(1200, 400, 20_000_000),
            support_model="dedicated",
            sdk_targets=["python", "typescript", "go", "java"],
        )
        return [starter, growth, enterprise]

    def recommend_portal_assets(self, tiers: Iterable[ProductTier]) -> list[PortalAsset]:
        assets: list[PortalAsset] = []
        for tier in tiers:
            assets.extend(
                [
                    PortalAsset(
                        asset_type="quickstart",
                        title=f"{tier.code.title()} Quickstart",
                        audience="new developer",
                        related_tier=tier.code,
                    ),
                    PortalAsset(
                        asset_type="pricing-guide",
                        title=f"{tier.code.title()} Usage and Limits",
                        audience="buyer",
                        related_tier=tier.code,
                    ),
                    PortalAsset(
                        asset_type="sample-app",
                        title=f"{tier.code.title()} Reference Application",
                        audience="implementer",
                        related_tier=tier.code,
                    ),
                ]
            )
        return assets

    def summarize_value(self) -> dict[str, int]:
        totals: dict[str, int] = {}
        for endpoint in self.endpoints:
            totals.setdefault(endpoint.category, 0)
            totals[endpoint.category] += endpoint.monetization_weight
        return totals


endpoints = [
    Endpoint("ListAccounts", "GET", "/v1/accounts", 1, 3, "core-data"),
    Endpoint("RunInsights", "POST", "/v1/insights", 3, 5, "analytics"),
    Endpoint("GeneratePortal", "POST", "/v1/portal", 2, 4, "dx-automation"),
    Endpoint("OptimizePricing", "POST", "/v1/pricing", 3, 4, "revenue-intel"),
    Endpoint("PublishCampaign", "POST", "/v1/campaigns", 2, 3, "distribution"),
]

designer = ApiProductDesigner(endpoints)
tiers = designer.build_tiers()
assets = designer.recommend_portal_assets(tiers)

for tier in tiers:
    print(
        tier.code,
        tier.monthly_price,
        tier.policy.requests_per_minute,
        ",".join(tier.included_categories),
    )

for asset in assets[:4]:
    print(asset.asset_type, asset.title, asset.related_tier)
```

## AC3: SaaS Growth Engine

**Purpose:** Convert product usage, content engagement, pricing posture, and churn risk into autonomous growth loops.

**Design Pattern:** Growth Feedback Lattice — acquisition, activation, expansion, and retention signals continuously reshape content, offers, onboarding, and packaging.

### Detailed Description

AC3 is where the superskill proves that growth is not a marketing afterthought but a systems property. It
watches how prospects discover content, how developers evaluate the API, how teams convert into paid
workspaces, and how account behavior changes after onboarding. Those signals do not live in separate
dashboards here. AC3 fuses them into one growth graph so that a weak activation metric can trigger tutorial
redesign, a strong feature adoption curve can trigger upsell messaging, and a rising churn cohort can reopen
pricing or documentation interventions before revenue is lost. It operates on the full lifecycle, not just
the top of funnel.

The reason this module requires all four forges is that growth depends on persuasion, product design,
interface contracts, and educational clarity at once. Content intelligence reveals which topics and
narratives bring in qualified traffic. SaaS intelligence understands onboarding checkpoints, entitlement
progression, expansion paths, and churn indicators. API intelligence explains which usage events constitute
real activation and which technical constraints block adoption. Documentation intelligence clarifies whether
failure to convert is caused by unclear instruction rather than weak demand. AC3 can distinguish those cases
because it sees the entire chain from first click to recurring revenue event.

AC3 also changes the timescale of growth work. Traditional organizations wait for quarterly readouts before
adjusting campaigns or pricing. This module compresses that into ongoing loops. Cohort decay can trigger a
nurture sequence update within hours. A documentation page that produces unusually strong sandbox creation
can be promoted into a landing page and social campaign. A usage threshold that correlates with multi-seat
expansion can be turned into a plan boundary and customer-success motion. Instead of reporting what
happened, AC3 continuously rewires the system so tomorrow's behavior is better than today's.

The emergent capability is autonomous commercial adaptation. AC3 does not simply analyze churn or optimize
copy. It can redesign the growth loop by changing messaging, onboarding sequence, feature exposure, and
commercial packaging in a single cascade. That requires a fused understanding of content assets, SaaS
metrics, API behaviors, and documentation pathways. Without that fusion, growth teams can only optimize
fragments. With AC3, the platform can learn its own revenue mechanics and intervene before human teams even
convene.

### APEX-Only Behaviors

- Links content attribution to activation and expansion instead of stopping at lead volume.
- Treats onboarding docs, in-product moments, and pricing copy as adjustable growth levers in the same
  loop.
- Turns usage milestones into both upsell triggers and editorial insights for future acquisition
  content.
- Predicts churn from blended product, support, documentation, and campaign signals rather than siloed
  CRM data.
- Builds growth motions that are technically feasible because they are tied to real API and
  entitlement behavior.

### Key Operations

- Ingest traffic, signup, activation, feature usage, support, and billing events into one cohort
  model.
- Score acquisition channels by downstream conversion quality rather than raw volume.
- Predict churn using product engagement, documentation completion, and usage trajectory signals.
- Select interventions such as nurture sequences, pricing prompts, tutorial revisions, or feature flag
  exposure.
- Surface expansion moments when usage patterns indicate team adoption or enterprise fit.
- Recommend experiments that connect channel messaging to product outcomes and subscription changes.
- Publish growth narratives back into AC5 and revenue insights back into AC6.

### Code Example — Growth loop that scores churn, selects interventions, and links content sources to recurring revenue outcomes

```python
from __future__ import annotations

from dataclasses import dataclass
from statistics import mean
from typing import Iterable


@dataclass(slots=True)
class AccountSignal:
    account_id: str
    acquisition_source: str
    docs_completion_rate: float
    activation_events: int
    api_calls_last_30d: int
    team_members: int
    plan_value: int
    support_tickets_last_30d: int


@dataclass(slots=True)
class Intervention:
    account_id: str
    label: str
    rationale: str
    priority: str


class GrowthEngine:
    def __init__(self, accounts: Iterable[AccountSignal]) -> None:
        self.accounts = list(accounts)

    def churn_score(self, account: AccountSignal) -> float:
        score = 0.0
        score += 0.35 if account.docs_completion_rate < 0.4 else 0.05
        score += 0.30 if account.activation_events < 3 else 0.04
        score += 0.20 if account.api_calls_last_30d < 2_000 else 0.06
        score += 0.10 if account.support_tickets_last_30d > 4 else 0.02
        score += 0.05 if account.team_members == 1 else 0.01
        return min(score, 1.0)

    def expansion_score(self, account: AccountSignal) -> float:
        value_density = account.api_calls_last_30d / max(account.plan_value, 1)
        team_factor = 0.25 if account.team_members >= 5 else 0.08
        docs_factor = 0.15 if account.docs_completion_rate >= 0.7 else 0.03
        usage_factor = 0.35 if value_density > 40 else 0.12
        activation_factor = 0.25 if account.activation_events >= 6 else 0.06
        return min(team_factor + docs_factor + usage_factor + activation_factor, 1.0)

    def recommend(self) -> list[Intervention]:
        recommendations: list[Intervention] = []
        for account in self.accounts:
            churn = self.churn_score(account)
            expansion = self.expansion_score(account)
            if churn >= 0.65:
                recommendations.append(
                    Intervention(
                        account.account_id,
                        "retention-recovery",
                        "Low docs completion and weak activation imply education-led churn risk.",
                        "high",
                    )
                )
            elif expansion >= 0.70:
                recommendations.append(
                    Intervention(
                        account.account_id,
                        "usage-upsell",
                        "High usage density and team growth support proactive plan expansion.",
                        "high",
                    )
                )
            else:
                recommendations.append(
                    Intervention(
                        account.account_id,
                        "nurture-optimization",
                        "Stable account; improve messaging and onboarding to increase depth of use.",
                        "medium",
                    )
                )
        return recommendations

    def source_quality(self) -> dict[str, float]:
        grouped: dict[str, list[float]] = {}
        for account in self.accounts:
            grouped.setdefault(account.acquisition_source, []).append(
                self.expansion_score(account) - self.churn_score(account)
            )
        return {source: round(mean(scores), 3) for source, scores in grouped.items()}
```

### Code Example — Revenue model that joins content, docs, usage, and billing metrics into pricing and growth recommendations

```python
from __future__ import annotations

from dataclasses import dataclass
from statistics import mean
from typing import Iterable


@dataclass(slots=True)
class RevenueRecord:
    account_id: str
    source: str
    content_cluster: str
    docs_path: str
    api_calls: int
    plan_price: int
    expansion_value: int
    activated: bool
    converted: bool


class RevenueIntelligence:
    def __init__(self, records: Iterable[RevenueRecord]) -> None:
        self.records = list(records)

    def funnel(self) -> dict[str, float]:
        total = len(self.records)
        activated = sum(1 for record in self.records if record.activated)
        converted = sum(1 for record in self.records if record.converted)
        return {
            "activation_rate": round(activated / total, 3) if total else 0.0,
            "conversion_rate": round(converted / total, 3) if total else 0.0,
        }

    def revenue_by_cluster(self) -> dict[str, int]:
        totals: dict[str, int] = {}
        for record in self.records:
            totals.setdefault(record.content_cluster, 0)
            totals[record.content_cluster] += record.plan_price + record.expansion_value
        return totals

    def docs_value(self) -> dict[str, float]:
        grouped: dict[str, list[int]] = {}
        for record in self.records:
            grouped.setdefault(record.docs_path, []).append(record.api_calls)
        return {path: round(mean(values), 1) for path, values in grouped.items()}

    def pricing_recommendation(self) -> str:
        avg_calls = mean(record.api_calls for record in self.records)
        avg_price = mean(record.plan_price for record in self.records)
        if avg_calls > 140_000 and avg_price < 400:
            return "Raise growth quota price or introduce a higher metered tier."
        if avg_calls < 10_000 and self.funnel()["conversion_rate"] < 0.10:
            return "Lower activation barrier before repricing; conversion problem is upstream."
        return "Current pricing posture is healthy; iterate through targeted experiments."

    def weak_link(self) -> str:
        activation = self.funnel()["activation_rate"]
        conversion = self.funnel()["conversion_rate"]
        if activation < 0.45:
            return "Developer activation is the primary bottleneck."
        if conversion < 0.12:
            return "Commercial packaging is weaker than product engagement."
        return "Top of funnel and monetization are balanced; focus on expansion."
```
