## Module CQ1: Intent Intelligence Engine

**Purpose**: Transform raw keyword data into actionable content blueprints by fusing keyword clusters with search intent classification and content gap analysis.

**Source Skills**: `seo-keyword-strategist` + `seo-content-refresher` (gap detection)

### Key Operations

1. **Keyword Universe Expansion**
   - Seed keyword → cluster generation → intent classification
   - Output: Prioritized keyword map with difficulty, volume, and intent tags

2. **Search Intent Mapping**
   - Classify every keyword as: Informational / Navigational / Commercial / Transactional
   - Map intent to content format (info → blog, commercial → comparison, transactional → landing page)

3. **Content Gap Scoring**
   - Cross-reference existing content inventory against keyword clusters
   - Score gaps: `gap_score = (search_volume × intent_match) / (competition × existing_coverage)`

4. **Feedback Intake Port** ← CQ6
   - Receive underperforming content IDs with diagnosis data
   - Re-evaluate keyword targeting for flagged content
   - Inject new keyword opportunities from CQ8 competitive radar

### Example: SaaS Onboarding Content

```yaml
seed_keyword: "project management software"
clusters:
  - cluster: "project management for small teams"
    volume: 8,100/mo
    difficulty: 34
    intent: commercial_investigation
    format: comparison_guide
    gap_score: 0.87  # HIGH — no existing content covers this
    
  - cluster: "how to manage remote projects"
    volume: 4,400/mo
    difficulty: 22
    intent: informational
    format: long_form_guide
    gap_score: 0.62  # MEDIUM — existing blog partially covers

  - cluster: "project management tool pricing"
    volume: 12,000/mo
    difficulty: 61
    intent: transactional
    format: pricing_landing_page
    gap_score: 0.94  # CRITICAL — no pricing comparison content

feedback_intake:
  flagged_content_id: "blog-047"
  diagnosis: "ranking_decay_-23_positions_90d"
  action: re_cluster_and_retarget
```

## Module CQ2: Content Synthesis Reactor

**Purpose**: Generate multi-format content from a single brief, fusing SEO writing discipline with creative content formats to produce blog posts, video scripts, guides, infographic briefs, and podcast outlines simultaneously.

**Source Skills**: `seo-content-writer` + `content-creator`

### Key Operations

1. **Brief Ingestion**
   - Accept structured brief from CQ1 (keyword cluster, intent, format, gap score)
   - Determine primary format + derivative formats

2. **SEO Content Generation**
   - Primary keyword placement: title, H1, first 100 words, meta description
   - Entity salience optimization: identify and distribute related entities
   - Semantic topical coverage: ensure all subtopics in the cluster are addressed
   - Internal link mapping: connect to existing content inventory

3. **Multi-Format Derivation**
   - From one research pass, generate all derivative formats:

   ```
   Primary Brief ──┬──▶ Long-Form Blog Post (2,500+ words)
                    ├──▶ Video Script (8-12 min)
                    ├──▶ Infographic Brief (key stats + flow)
                    ├──▶ Podcast Episode Outline (talking points)
                    └──▶ Slide Deck Framework (10-15 slides)
   ```

4. **NLP Signal Injection**
   - Flesch-Kincaid target: 55-65 (accessible but authoritative)
   - Sentence variety: mix short (≤8 words) and complex (20-30 words)
   - Transition density: ≥4% of sentences begin with transitions
   - Question insertion: 1 question per 300 words for engagement

## Module CQ3: Persuasion Architecture

**Purpose**: Layer persuasion psychology across all content formats — engineering hooks, story structures, offers, and CTAs that convert readers into leads and leads into customers.

**Source Skills**: `copywriting`

### Key Operations

1. **Hook Engineering**
   - Pattern-interrupt openers calibrated to platform and format
   - Hook types mapped to intent:

   | Intent | Hook Type | Example Pattern |
   |---|---|---|
   | Informational | Curiosity Gap | "Most teams get [topic] wrong. Here's why..." |
   | Commercial | Social Proof | "[Company] grew 340% after switching. Here's how..." |
   | Transactional | Urgency + Scarcity | "Only available until [date]. 2,400 teams already joined." |
   | Navigational | Authority | "The official guide from [brand], updated for 2025." |

2. **Story Architecture (AIDA × PAS)**
   - Attention → Interest → Desire → Action (macro structure)
   - Problem → Agitation → Solution (within each section)
   - Every content piece follows: Hook → Story → Mechanism → Offer → CTA

3. **CTA Engineering**
   - Primary CTA: aligned with content intent
   - Secondary CTA: lower commitment fallback
   - Micro-CTAs: embedded throughout content (not just at the end)

   ```
   Informational Post:
     Primary CTA:   "Download the free checklist" (lead magnet)
     Secondary CTA: "Subscribe for weekly tips"
     Micro-CTA:     "Bookmark this section for later"

   Commercial Post:
     Primary CTA:   "Start your free trial"
     Secondary CTA: "See pricing plans"
     Micro-CTA:     "Compare features side-by-side"
   ```

4. **Conversion Copy Patterns**
   - Benefit-first headlines (not feature-first)
   - Future pacing: paint the after-state before explaining the process
   - Objection handling embedded in body copy
   - Power words distribution: aim for 2-3 per headline, 1 per subheading

### Example: Landing Page Persuasion Stack

```yaml
page_type: saas_trial_signup
target_persona: "Marketing Manager, 30-45, B2B SaaS"

hook:
  type: social_proof_with_specificity
  copy: "12,847 marketing teams ship content 3× faster with [Product]"
  
story:
  problem: "You spend 40 hours/week creating content that gets buried on page 3"
  agitation: "Meanwhile, your competitors publish daily and steal your rankings"
  solution: "An AI-powered content engine that handles research → writing → optimization"

mechanism:
  step_1: "Paste your keyword → get a full content brief in 30 seconds"
  step_2: "Review the AI draft → edit with real-time SEO scoring"
  step_3: "Publish → track → auto-optimize based on performance data"

offer:
  primary: "14-day free trial, no credit card required"
  guarantee: "Cancel anytime. Export all your content if you leave."
  
cta_stack:
  primary: { text: "Start Creating Better Content →", color: "green", position: "hero" }
  secondary: { text: "See a 3-minute demo", position: "below_fold" }
  exit_intent: { text: "Get our free SEO checklist before you go", type: "popup" }

objection_handlers:
  - objection: "AI content sounds generic"
    response: "Our AI trains on YOUR brand voice. See examples from [similar company]."
  - objection: "We already have a content team"
    response: "Teams using [Product] produce 3× more content with the same headcount."

## Module CQ4: Distribution Orchestrator

**Purpose**: Adapt content for platform-specific distribution, schedule publication across channels, and orchestrate amplification sequences that maximize reach per content asset.

**Source Skills**: `social-content` + `marketing-automation`

### Key Operations

1. **Platform-Specific Adaptation**
   - One content piece → multiple platform-native versions:

   ```
   Blog Post (2,800 words)
     ├──▶ LinkedIn Article     (1,200 words, professional tone, no emojis)
     ├──▶ Twitter/X Thread     (8-12 tweets, hooks per tweet, visual per 3rd tweet)
     ├──▶ Instagram Carousel   (10 slides, key insight per slide, swipe CTA)
     ├──▶ YouTube Description  (timestamps, keywords, links, chapters)
     ├──▶ TikTok Script        (60 sec, pattern interrupt opener, text overlay cues)
     ├──▶ Newsletter Excerpt   (400 words, curiosity gap, link to full post)
     └──▶ Reddit Post          (value-first, no self-promotion, community-native)
   ```

2. **Publication Scheduling**
   - Optimal posting times per platform (data-driven, adjusted by CQ5 analytics)
   - Staggered release: blog first (indexing), social same day, email 24h later
   - Evergreen re-queue: high-performing content re-enters rotation every 90 days

3. **Amplification Sequences**
   - Day 0: Publish blog + LinkedIn article
   - Day 1: Twitter thread + Instagram carousel
   - Day 2: Email to segment (via CQ7)

   - Day 7: Retarget engaged non-converters with variant hook
   - Day 14: Re-share top performer with new angle
   - Day 30: Evaluate via CQ5 → CQ6 feedback decision

4. **Cross-Channel Attribution Tagging**
   - Auto-append UTM parameters per platform and campaign
   - Generate unique tracking links for each distribution channel
   - Feed attribution data to CQ5 for unified measurement

**Source Skills**: `analytics-tracking` + `data-storytelling`

### Key Operations

1. **Unified Metrics Dashboard**
   - Aggregate data across blog, social, email, and paid channels
   - Normalize metrics to comparable units:

   | Channel | Reach Metric | Engagement Metric | Conversion Metric |
   |---|---|---|---|
   | Blog | Pageviews, Sessions | Time on Page, Scroll Depth | Form Fills, Trial Starts |
   | LinkedIn | Impressions | Reactions, Comments, Shares | Profile Clicks, Link Clicks |
   | Twitter/X | Impressions | Likes, Retweets, Replies | Link Clicks, Profile Visits |
   | Email | Delivered | Open Rate, Click Rate | Reply Rate, Conversion Rate |
   | Instagram | Reach | Saves, Shares, Comments | Link Clicks (bio), DMs |

2. **Multi-Touch Attribution**
   - Track the full journey: first touch → assisted touches → converting touch
   - Attribution models supported:
     - **Linear**: equal credit across all touchpoints
     - **Time-decay**: more credit to recent touchpoints
     - **Position-based**: 40% first, 40% last, 20% middle
     - **Data-driven**: algorithmic (uses CQ5 historical data)

3. **Content Performance Scoring**
   - Composite score per content piece:
   ```
   performance_score = (

     0.25 × engagement_score +
     0.25 × conversion_score +

   - Scores normalized 0-100; threshold for CQ6 refresh trigger: < 40

4. **Data → Narrative Translation**
   - Transform raw metrics into actionable insights
   - Auto-generate weekly content performance reports
   - Chart selection logic: trends → line, comparison → bar, composition → pie, relationship → scatter

## Module CQ6: Feedback Loop Engine

**Purpose**: Close the loop. Analyze performance data from CQ5, diagnose underperformance, trigger content optimization, and feed insights back into CQ1 and CQ2 for autonomous re-generation.

**Source Skills**: `seo-content-refresher` + `data-storytelling`

> **🔄 THIS IS THE EMERGENT PROPERTY**: CQ6 is the module that makes the entire
> system self-optimizing. Without it, you have 7 disconnected tools. With it,
> you have an autonomous content organism.

### Key Operations

1. **Decay Detection**
   - Monitor all published content for ranking and traffic decay
   - Decay signals:
     - Ranking drop > 10 positions over 30 days
     - Organic traffic decline > 25% month-over-month
     - Performance score drops below 40 threshold
     - Competitor publishes superior content on same keyword (via CQ8)

2. **Root Cause Diagnosis**
   - For each flagged content piece, diagnose WHY it's underperforming:

   | Symptom | Likely Cause | Module Fix |
   |---|---|---|
   | Ranking drop, traffic stable | Algorithm shift or competitor surge | CQ1: re-cluster keywords |
   | High traffic, low conversion | Weak CTA or intent mismatch | CQ3: re-engineer persuasion |
   | Low traffic, high conversion | Under-promoted or poor SEO | CQ4: re-distribute |
   | Low traffic, low conversion | Content quality or topic mismatch | CQ2: regenerate content |
   | Email open ↓, click stable | Subject line fatigue | CQ7: A/B new subject lines |
   | Social reach ↓ | Algorithm change or hook fatigue | CQ4: new platform hooks |

3. **Optimization Action Dispatch**
   - Based on diagnosis, dispatch to the appropriate module:
   ```
   CQ6 Diagnosis Result
     ├── keyword_mismatch    → dispatch to CQ1 (re-discover)
     ├── content_quality     → dispatch to CQ2 (re-generate)
     ├── persuasion_failure  → dispatch to CQ3 (re-architect CTAs)
     ├── distribution_gap    → dispatch to CQ4 (re-distribute)
     ├── email_fatigue       → dispatch to CQ7 (re-sequence)
     └── competitive_threat  → dispatch to CQ8 (re-analyze)
   ```

4. **Feedback Data Packet** → CQ1
   - Package sent back to CQ1 to close the loop:
   ```yaml
   feedback_packet:
     content_id: "blog-023"
     current_performance_score: 31
     decay_type: "ranking_drop_severe"
     diagnosis: "competitor_published_superior_content"
     original_keyword_cluster: "email marketing automation"
     recommended_action: "re_cluster_with_long_tail_variants"
     competitor_content_url: "https://competitor.com/email-automation-guide"
     competitive_gap_data: # from CQ8
       missing_subtopics: ["AI personalization", "deliverability optimization"]
       word_count_gap: "+1,200 words"
       backlink_gap: "+34 referring domains"
   ```

### Example: Autonomous Refresh Cycle

## Module CQ7: Email Nurture Pipeline

**Purpose**: Build automated email sequences that segment audiences, trigger personalized drip campaigns, A/B test messaging, and optimize send cadence based on CQ5 engagement data.

**Source Skills**: `email-sequence` + `marketing-automation`

### Key Operations

1. **Audience Segmentation**
   - Segment by behavior, not just demographics:
     - Content consumers (read 3+ articles)
     - Lead magnet downloaders (specific topic interest)
     - Trial users (product-aware, need activation)
     - Churned users (re-engagement candidates)
     - High-intent visitors (pricing page viewers)

2. **Trigger-Based Sequences**
   - Event-driven emails, not calendar-driven:
   ```
   TRIGGER: Downloaded "Remote PM Checklist"
     ├── Day 0: Deliver checklist + "here's how teams use this"
     ├── Day 2: Case study — "How [Company] used this checklist to ship 2× faster"
     ├── Day 5: Educational — "3 remote PM mistakes even experienced managers make"
     ├── Day 8: Social proof — "12,000 teams trust [Product] for remote PM"
     ├── Day 12: Offer — "Start your free trial (exclusive 21-day extended trial)"
     └── Day 15: Urgency — "Your extended trial offer expires in 48 hours"
   
   BRANCH LOGIC:
     If opened Day 5 email → skip to Day 8 (engaged, accelerate)
     If not opened Day 2 → resend Day 2 with new subject line
     If clicked Day 12 CTA → exit sequence (entered trial funnel)
   ```

3. **A/B Testing Framework**
   - Test one variable per experiment:
     - Subject lines: curiosity gap vs. benefit-driven vs. question
     - Send times: morning (8am) vs. lunch (12pm) vs. evening (7pm)
     - CTA copy: action verb vs. benefit statement vs. urgency
     - Email length: short (150 words) vs. medium (300) vs. long (500+)
   - Statistical significance threshold: 95% confidence, min 1,000 sends per variant

4. **Performance Optimization** ← CQ5
   - Receive engagement data from CQ5 analytics
   - Auto-adjust: send times, subject line patterns, sequence length
   - Kill underperforming sequences (open rate < 15% after 3 sends)

## Module CQ8: Competitive Radar

**Purpose**: Continuously monitor the competitive content landscape — detect SERP movements, identify content gaps, score opportunities, and feed competitive intelligence into CQ1 (new keywords) and CQ6 (refresh triggers).

**Source Skills**: `seo-keyword-strategist` (SERP analysis) + `seo-content-refresher` (competitive gap detection)

### Key Operations

1. **SERP Movement Tracking**
   - Monitor target keyword rankings daily
   - Alert on: position drops > 5, new competitors entering top 10, featured snippet changes
   - Track competitor content freshness (last updated dates, content length changes)

2. **Competitive Content Gap Analysis**
   - For each target keyword cluster, compare:

   | Dimension | Our Content | Top Competitor | Gap |
   |---|---|---|---|
   | Word Count | 2,200 | 4,500 | -2,300 words |
   | Subtopics Covered | 8/12 | 11/12 | -3 subtopics |
   | Backlinks | 12 | 67 | -55 referring domains |
   | Content Freshness | 8 months | 2 weeks | STALE |
   | Multimedia | 2 images | 6 images + video | -4 images, -1 video |
   | Structured Data | FAQ schema | FAQ + HowTo + Video | Missing 2 schema types |

3. **Opportunity Scoring**
   - Score each gap as an opportunity:
   ```
   opportunity_score = (
     0.35 × keyword_volume_potential +
     0.25 × competitive_weakness +
     0.20 × our_existing_authority +
     0.10 × content_effort_inverse +
     0.10 × conversion_potential
   )
   ```
   - Opportunities scoring > 70 are auto-dispatched to CQ1

4. **Competitive Intelligence Feeds**
   - → CQ1: New keyword opportunities discovered from competitor content
   - → CQ6: Competitor superiority triggers content refresh
   - → CQ3: Competitor CTA patterns worth testing

## Cross-Module Integration — The Autonomous Feedback Loop

The following integration patterns show how modules interact to create the emergent self-optimizing behavior. No single module operates in isolation.

### Pattern 1: The Ω Loop (Primary Feedback Cycle)

```
CQ1 (discover) → CQ2 (create) → CQ3 (persuade) → CQ4 (distribute)
                                                          │
                                                          ▼
CQ1 (re-discover) ← CQ6 (optimize) ← CQ5 (measure) ← [audience]
```

**Trigger**: CQ5 detects performance score < 40
**Action**: CQ6 diagnoses → dispatches fix to CQ1/CQ2/CQ3/CQ4
**Result**: Content autonomously improves without human intervention
**Cycle time**: 7-30 days per iteration

### Pattern 2: The Competitive Injection

```
CQ8 (radar) ──┬──▶ CQ1 (new keyword opportunities)
               └──▶ CQ6 (competitor-triggered refresh)
```

**Trigger**: CQ8 detects competitor ranking gain or new entrant
**Action**: Injects competitive intelligence into discovery (CQ1) and refresh (CQ6)
**Result**: Content strategy adapts to competitive landscape automatically

### Pattern 3: The Nurture Amplifier

```
CQ5 (analytics) ──▶ CQ7 (email optimization)
CQ7 (email data) ──▶ CQ5 (unified metrics)
```

**Trigger**: CQ5 identifies high-performing content with low email distribution
**Action**: CQ7 creates targeted sequence for that content
**Result**: Under-leveraged content gets email amplification

### Pattern 4: The Full Autonomous Sequence

```
Week 1:  CQ8 detects competitor gap → CQ1 discovers new keyword cluster
Week 2:  CQ1 creates brief → CQ2 generates content → CQ3 adds persuasion
Week 3:  CQ4 distributes across channels → CQ7 launches email sequence
Week 4:  CQ5 measures performance → data flows to CQ6
Week 5:  CQ6 evaluates — if score > 60, amplify; if < 40, refresh
Week 6+: Loop continues — each cycle improves content quality
```
